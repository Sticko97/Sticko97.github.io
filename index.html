Activity #1: Explain what is Blue/Green deployment

Blue/Green deployment is an application release model that transfers user traffic in increments. The user traffic is obtained from the previous version of an app or microservice of a similar or nearly identical new release, both running in production.

Old version is Blue
New version is Green

Once data is fully transferred from Blue to Green, Blue can be thought of as a snapshot, a precaution that allows us to rollback data to a past version.

Activity #2: List out 3 remote host solutions with a short explanation

Remote hosting is a method for delivering software applications, databases, and files to users from remote servers that are housed off-site in data centers.

  1. AWS: Offers a scalable and adaptable infrastructure for hosting data, other services, and web applications.
  2. Azure: Provides a variety of cloud services, including virtual machines, databases, AI and machine learning tools, and services geared at developers, such as Azure DevOps.
  3. Digital Ocean: Provides cloud computing resources including managed databases, Kubernetes, and virtual private servers (Droplets).

Activity #3: Explain what is Rolling deployment

A rolling deployment is a deployment technique that totally updates the infrastructure that the program is operating on in order to gradually replace older versions of an application with newer versions.

Activity #4: Explain what is Canary deployment

A canary deployment is a gradual rollout of an application that divides traffic between an existing version and a new version, pushing it out to a select group of users before rolling it out to all users.

Activity #5: Explain what is In-place deployment

Allows you to deploy your application without building new infrastructure; however, during these deployments, your application's availability may be impacted. Additionally, this strategy reduces the administrative burden and infrastructural expenditures related to generating new resources.
